오늘의 작업 내역을 docs/history.md에 기록해줘.

## 진행 순서

**1단계 — 오늘 날짜 확인**
오늘 날짜는 `$CURRENT_DATE` 형식 (예: 2026-05-02) 으로 사용한다.

**2단계 — 오늘 작업 내역 파악**
다음 소스를 모두 참고해서 오늘 실제로 한 작업을 파악한다:
- `git log --since="today" --oneline` 으로 오늘 커밋 내역
- `git diff --name-only HEAD~5..HEAD` 로 최근 변경 파일 목록
- 이 대화에서 오늘 논의하거나 작업한 내용

**3단계 — docs/history.md 읽기**
`docs/history.md` 전체를 읽고, 오늘 날짜(`## YYYY-MM-DD` 형식)로 시작하는 섹션이 있는지 확인한다.

**4단계 — 작성 or 업데이트**

- **오늘 날짜 섹션이 없으면**: `docs/history.md` 맨 아래에 아래 형식으로 추가한다.
- **오늘 날짜 섹션이 이미 있으면**: 기존 항목을 읽고, 누락된 작업만 추가하거나 잘못된 내용을 수정한다. 이미 있는 항목은 중복 추가하지 않는다.

## 추가할 형식

```
---

## YYYY-MM-DD

- 작업내역1
- 작업내역2
- 작업내역3
```

## 작업내역 작성 규칙

- 각 항목은 `- `로 시작하는 한국어 한 줄 요약
- 파일명/경로는 백틱으로 감싸기 (예: `src/app/zodiac/page.tsx`)
- 기능 단위로 묶어서 작성 (파일별 나열 X, 기능별 요약 O)
- 예시:
  - `별자리 운세 페이지 생성 (src/app/zodiac/page.tsx) — 12별자리 선택 UI, 상세 정보 표시`
  - `타로 원카드 페이지 생성 — Framer Motion 카드 플립 애니메이션, 78장 랜덤 뽑기`
  - `fortunes.json 운세 18개 → 30개 확장, difficulty 필드 추가`
